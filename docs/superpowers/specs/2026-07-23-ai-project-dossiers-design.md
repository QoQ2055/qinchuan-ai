# AI 项目档案卡补充与背景图规格

## 目标

在“项目与资历”区置顶三项 2026 年 AI 制作项目，突出秦川的全流程制作能力、已落地成果与代表作；同时把两张既有 AI 分镜图作为低透明度底图，以更紧凑的电影制作档案风格补足版面留白。

## 内容与信息层级

新增三张项目卡，均位于既有设计项目之前，按以下顺序呈现：

1. **《我爱科学》AI 动画化及行业内容合作** — 全流程制作，2026.03 - 至今。正文合并中少社《我爱科学》、中国金融传媒及云南文旅三个合作方向；强调内容创意、流程优化、金融消保/反诈/营销动画与文旅项目实施状态。
2. **AI 课本动画化** — 全流程制作，2026.03 - 至今。强调人教/沪教双版本儿童英语项目，IMAGE2、Stable Diffusion 2.0、AI 口型同步与 TTS 的统一制作流程，以及单人日产 2 条和教研认可。
3. **AI 仿真人短剧开发** — 全流程制作，2026.03 - 至今。强调选题、剧本、生成、剪辑、上线发行的全流程，以及红果短剧稳定运营和三部代表作。

每张卡保留现有“时间 / 角色”荧光元信息、标题和正文结构；不删减用户提供的事实，仅将长段落整理为易扫读的作品集语句。

## 视觉规格

- 三张置顶卡使用既有 `.project-dossier` 模式，桌面两列、手机单列。
- 两张现有分镜图作为前两张新卡的 CSS 背景；第三张短剧卡使用导演监看图的裁切变体或无图深色卡，以避免同屏重复。
- 图像呈灰度、对比度略提升；背景透明度上限为 **25%**，并添加不透明度约 **75% 以上**的深色渐变遮罩。
- 文字、荧光日期及标签位于遮罩之上；背景不得降低正文对比度或改变移动端单列阅读。
- 鼠标悬停仅允许现有边框色反馈，不添加位移、缩放或复杂动画。

## 验证

- 渲染测试应包含三项新项目名称、三项“全流程制作”元信息，以及三部短剧代表作。
- CSS 测试应确认带图卡具有深色遮罩/背景图规则，且移动端仍为单列。
- 运行完整测试、GitHub Pages 导出，确认静态产物引用图片并可在公开链接加载。
