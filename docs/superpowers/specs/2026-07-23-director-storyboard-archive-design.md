# 导演分镜档案视觉加密规格

## 目标

在不牺牲简历可读性的前提下，为秦川的线上简历补充导演作品集的视觉层次。页面保持克制的黑色电影档案基调；用 AI 动画分镜图像组织章节节奏，而非将一张叙事画面铺满整页。

## 设计决定

- **全站背景**：保留纯黑基底，仅增加极低对比的胶片颗粒／网格纹理。不得使用人物或分镜画面作全页重复背景，以免长文本、移动端裁切和内容对比度失控。
- **两张新增图像**：生成两张横向 16:9 的无文字 AI 动画分镜档案图。
  1. **脚本到镜头**：纸本分镜、角色设定、时间轴、工作台与监看屏，表现从创意到镜头的制作起点。
  2. **角色、场景与节奏**：多格动画分镜、镜头标记、色彩脚本与剪辑轨道，表现生产流程与交付节奏。
- **章节编排**：
  - 第一张作为工作经历与项目区之间的“制作档案 / 01”横向过场：左侧为短标题及一句方法论，右侧为图像。
  - 第二张置于项目卡之后、教育经历之前，采用镜像排版作为“制作档案 / 02”过场：左侧图像、右侧短标题和说明。
  - 两个过场均有细灰线和极小荧光黄索引，不添加额外装饰线或夸张动画。
- **现有项目图**：现有两张 AI 图不再承担主视觉责任；继续作为前两张项目卡的低对比氛围背景，透明度不超过 18%，并以深色遮罩保证正文对比度。
- **互动与动效**：过场图在支持悬停的设备上只做 1–2% 的缓慢缩放及边框亮度变化；在 `prefers-reduced-motion` 下完全静止。移动端不触发悬停动效。

## 排版与可访问性

- 过场最大宽度沿用现有内容网格，避免引入全宽海报造成页面空洞或阅读跳跃。
- 桌面端使用 5:7 或 7:5 的文字／图像比例；窄屏切换为单列，标题在上、图像在下，图片高度限制在 16:10 的安全比例。
- 图像只作补充；全部叙事信息仍以 HTML 文本呈现，并提供描述性 `alt`。
- 不在图像中生成可读文字、商标或水印，避免与页面的真实排版冲突。

## 验证

- 渲染测试验证两个制作档案过场、两张图像的 `alt` 与现有项目内容同时存在。
- CSS 测试验证：全站没有叙事图像作为 `body` 背景；项目卡背景透明度不超过 18%；过场在移动端为单列；减弱动画偏好关闭过渡。
- 完整构建、静态导出、GitHub Pages 发布后，在桌面及窄屏宽度检查图像路径、可读性和无横向溢出。
