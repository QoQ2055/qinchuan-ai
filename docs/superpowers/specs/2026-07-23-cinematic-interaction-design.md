# 克制电影档案交互与排版稳定规格

## 目标

修复短剧图片区与现有内容网格的对齐风险，并以不改变文档流尺寸的轻量交互增强导演作品档案的浏览节奏。

## 交互

- 顶部导航显示当前章节，并以 1px 荧光黄阅读进度线反映页面滚动进度。
- 标题、正文、经历条目、项目卡和图片区进入视口时只做一次 `opacity` 与最多 `translateY(12px)` 的揭示；动画不改变列宽、边距、高度或定位。
- 导航链接、经历条目与项目卡悬停时只改变颜色、边框与背景亮度；图像悬停最多 `scale(1.02)`。
- 首屏合作链接继续使用短箭头推进反馈，不增加鼠标跟随、自动播放、循环闪烁或沉重视差。

## 排版稳定

- 短剧视觉区作为项目网格内的完整跨列条目，`grid-column: 1 / -1`；图片接触印样使用独立内层网格，不能影响其他项目卡的列计算。
- 宽屏图片保持紧凑点缀尺寸；小于 760px 时内层网格单列，图片宽度回到容器宽度，且不产生横向溢出。
- 动效类通过 JavaScript 在元素进入视口时添加，默认初始状态保持可见，以免脚本失败导致内容隐藏。

## 可访问性与验证

- `prefers-reduced-motion: reduce` 下不执行揭示、进度过渡、图片缩放与任何新交互动画。
- 测试验证滚动进度元素、当前导航状态、revealed 类规则、短剧区跨列和窄屏单列规则。
- 构建、静态导出与公开页验证不得出现页面级横向溢出。
