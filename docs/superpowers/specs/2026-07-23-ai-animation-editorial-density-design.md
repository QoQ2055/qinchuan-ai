# AI 动画制作档案：视觉密度与可读性

## 目标

在不删减或改写简历正文的前提下，为页面补入克制的 AI 漫剧 / AI 动画视觉内容，并解决工作经历侧栏文字偏小的问题。

## 已确认方向

执行视觉稿的 **A + 03**：

- 采用“档案页插入式画面”，而不是覆盖整页或把每条履历做成卡片。
- 工作经历的日期与岗位文字放大，并继续与荧光黄悬停边线保持独立间距。

## 视觉与布局

1. 生成两张无文字、无水印的横向原创 AI 漫剧分镜插画：
   - 第一张置于“个人优势”与“工作经历”之间，表现从剧本、分镜到成片的制作管线。
   - 第二张置于“项目与资历”区之前，表现导演监看、镜头与动画资产协同。
2. 每张图以带细边框的“制作档案页”呈现，保留现有黑底、米白文字与荧光黄细节；桌面端形成两栏关系，移动端改为单栏。
3. 插画使用暗色手绘漫剧分镜质感，画面仅作为节奏与氛围补充，不压在正文文字下方。
4. 经历区的时间与岗位从当前小号注释提升到约 0.9rem，行高与颜色同步提升；移动端保持至少 0.88rem。

## 技术与质量约束

- 新图片保存在 `public/images/`，通过项目相对路径引用，确保 GitHub Pages 子路径部署可用。
- 使用 `loading="lazy"` 和明确的 `alt` 文本；装饰性图片不包含必要信息。
- 新增回归测试：确认两个视觉档案模块、图片路径、经历元信息字号和移动端字号存在。
- 验证构建、全部测试、GitHub Pages 导出与已发布页面。

## 不在范围内

- 不改动任何既有工作内容、项目内容、联系方式或主色调。
- 不新增复杂交互、视频播放或轮播组件。
